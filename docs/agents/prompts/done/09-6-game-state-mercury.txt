New interfaces — add to src/app/core/models/game-state.model.ts

export type MercurySlotStatus = 'locked' | 'available' | 'building' | 'operational';

export interface MercuryMinerState {
  poolCount: number;              // total miners in reserve (unassigned)
  assignments: Record<string, number>;  // slotId → number of miners assigned
}

export interface MercuryPath {
  id: string;
  segments: { col: number; row: number }[];  // empty for playtest
}

New mutations:
  selectMercuryZone(zoneId: string): void
    - Set _mercurySelectedZone(zoneId)
    - Load mercury-map.json starting zone seed slots from DataService
    - Set those slots + seedArea tiles to 'available' in _mercurySlotStates

  assignMiner(slotId: string): void
    - If poolCount > 0: poolCount--, assignments[slotId]++

  unassignMiner(slotId: string): void
    - If assignments[slotId] > 0: assignments[slotId]--, poolCount++

  reassignMiner(fromSlotId: string, toSlotId: string): void
    - If assignments[fromSlotId] > 0: assignments[fromSlotId]--, assignments[toSlotId]++

  setResourceReservation(resource: keyof ResourceStore, amount: number): void
    - Clamp amount >= 0
    - Update _resourceReservations

  setMercurySlotState(slotId: string, status: MercurySlotStatus): void
    - Update _mercurySlotStates

Slot unlock cascade:
  Existing placeMercuryBuilding() mutation should, when a building completes (status →
  'operational'), call a private _unlockAdjacentSlots(building: PlacedBuilding) method:
    - Find the slot at building.col/row via DataService.getMercuryMapData()
    - For each slotId in slot.adjacentTo: if current state is 'locked' → set to 'available'
  NOTE: placeMercuryBuilding currently sets status directly. If the build is completed via
  MercuryBuildService (via tick), the state transition happens in MercuryBuildService's
  completeBuild() method — add the cascade call there too.
