NEW MODEL SHAPE

Define exported interfaces/types using research vocabulary:

ResearchNode:
  id: string
  planet: string
  displayName: string
  category: 'capability' | 'efficiency' | 'reactive' | 'identity'
  tier: 1 | 2 | 3
  postV1?: boolean
  collaborative?: boolean
  durationYears: number
  description: string
  outcomeSummary: string[]
  unlockCondition: string
  prerequisites: string[]
  prerequisiteMode?: 'all' | 'any'
  spilloverPrerequisites: string[]
  spilloverGate?: { mode: 'min_naturalist_nodes'; count: number }
  effects: ResearchEffect[]
  arcIds?: string[]
  transfersFrom?: ResearchTransfer[]
  visualOrder?: number
  iconPath?: string

ResearchTransfer:
  fromNodeId: string
  effect: 'capability_gate' | 'queue_reduction' | 'time_reduction' | 'event_mitigation' | 'variance_reduction' | 'research_time_reduction'
  reason: string
  description: string

ResearchEffect:
  Keep existing TechEffect behavior but rename the type. Update effect names only where required by the new GDD:
  - remove rp_capacity_boost from new content; architect research should use flat research-time-reduction effects, not RP capacity
  - keep existing effect cases temporarily if old systems still compile, but mark RP boost as legacy during migration

ActiveResearch:

ResearchArcLogEntry:

DATA MIGRATION REQUIREMENTS

1. Canonical JSON file
   - public/data/research-tracks.json becomes the single canonical research graph file.
   - Existing public/data/tech-tree.json content is migrated into it.
   - The old small public/data/research-tracks.json entries should be merged as nodes or removed if they duplicate canonical GDD nodes.

2. Remove RP data from new nodes
   - Remove rpCost from the canonical node schema.
   - Keep durationYears.
   - Update player-facing copy from RP capacity language to slot/time language.

3. Full visible graph support
   - Every node must have displayName, tier, category, unlockCondition, description, outcomeSummary, durationYears, prerequisites, spilloverPrerequisites, effects.
   - Locked nodes can display title and unlockCondition from data.
   - Do not rely on silhouetteIconPath for hidden ??? nodes in the new system.

4. Moon visual order
   - Add visualOrder (or equivalent data-driven ordering) so Moon Phase 1 nodes render above/before Moon Phase 2 in the intended visual order. Do not depend only on recursive prerequisite depth.

5. Accessor rename
   - DataService should expose research-language methods:
     getResearchNode(id)
     getResearchNodesForPlanet(planetId)
     getAllResearchNodes()
   - Remove or temporarily deprecate getTechNode/getTechNodesForPlanet. If compatibility wrappers are needed, keep them only until 26-6.

6. Type aliases
   - Avoid broad permanent aliases like type TechNode = ResearchNode. If temporary aliases are required to keep this prompt small, add a clear TODO in 26-6 prompt scope and remove them there.
