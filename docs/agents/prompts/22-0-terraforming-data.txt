REQUIREMENTS

1. Per-planet phase structure (from terraforming-options.md)
   - Mars: Phase 1 Warming/Atmosphere → Phase 2 Nitrogen → Phase 3 Magnetic Protection → Phase 4 Water, plus the bio phase.

2. Choices per phase
   - Each phase offers the GDD's named CHOICES (e.g. Mars Phase 1: Orbital Laser Arrays / Statite Mirror / Polar Nuclear; Venus Phase 1: Dedicated Shade Mirror / Dyson Panel Redeployment; etc.). Each choice: id, name, description, cost (materials/components/Dyson draw as applicable), duration modifier, downstream effects, irreversibility flag, and any cross-planet flags (e.g. Venus nitrogen export → Mars Phase 2B).
   - Venus Opening Decision encoded as a special pre-Phase-1 choice with exactly two options (Wild / Europa Impact), with the day-length + ocean-coverage outcomes and the Europa irreversibility + Hunt-for-Life flag baked from the GDD numbers.

3. Boosts and Always-Available infrastructure
   - Per-phase optional BOOSTS (e.g. Vortex Engines, Electrochemical Atmosphere Processing) with cost + effect.
   - ALWAYS-AVAILABLE options (e.g. Venus Day/Night Orbital Mirrors, Mars Magnetic Protection add-anytime, Geothermal Drilling) as a separate category with prerequisites.

4. Phase events
   - Per-phase EVENT POOLS (positive/negative) with the GDD's events, each with: id, name, description, sign, trigger weighting, and the player RESPONSE OPTIONS (each option: label, cost, effect on timeline/state). The Venus two-moon-only [C] events are removed in the GDD — do not include them.

5. Extraction-rate / slider parameters
   - Encode the tunable extraction/allocation rates (e.g. Dyson shade % for Venus, delivery rates) as data with min/max/default so the UI sliders (22-2) and service are data-driven.

6. Visual-value inputs
   - Per phase, the target values for the derived visual/conditions readout (temperature, pressure, ocean %, composition) so the planet-panel conditions (19-3) and orrery texture cross-fade (Block 18/22) can interpolate by year. These are INPUTS to pure getValueAtYear() derivation — not stored animation state.
